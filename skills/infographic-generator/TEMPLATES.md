# 模板详细说明

本文档详细说明infographic-generator的6个核心模板及其使用方法。

## 模板概览

| 模板 | 名称 | 适用场景 | 默认风格 | 默认尺寸 |
|------|------|---------|---------|---------|
| `knowledge` | 知识科普 | 概念解释、知识分享、技术介绍 | 极简风 | 1920x1080 |
| `comparison` | 对比分析 | 产品对比、方案评估、优劣分析 | 商务风 | 1920x1080 |
| `process` | 流程说明 | 步骤指南、操作流程、工作流 | 扁平化 | 1920x1080 |
| `data` | 数据展示 | 数据报告、统计结果、调研分析 | 商务风 | 1200x800 |
| `xiaohongshu` | 小红书爆款 | 社交媒体分享、内容营销 | 马卡龙风 | 1080x1920 |
| `scientific` | 科研图表 | 学术论文、研究报告、实验数据 | 极简风 | 1200x800 |

---

## 1. 知识科普模板（knowledge）

### 适用场景
- 概念解释
- 知识分享
- 技术介绍
- 学习笔记
- 科普内容

### 内容结构

```
标题区
├─ 主标题（必填）
└─ 副标题（可选）

核心观点区（必填，2-7个要点）
├─ 要点1
│   ├─ 标题
│   ├─ 内容
│   └─ 图标
├─ 要点2
├─ 要点3
└─ ...

详细说明区（可选）
├─ 说明1
├─ 说明2
└─ ...

总结区（可选）
└─ 核心要点回顾
```

### JSON示例

```json
{
  "metadata": {
    "template": "knowledge",
    "style": "minimal"
  },
  "content": {
    "title": {
      "text": "Python编程语言",
      "subtitle": "简洁优雅的编程语言"
    },
    "sections": [
      {
        "id": "point-1",
        "type": "key-point",
        "title": "语法简洁",
        "content": "易学易用，代码可读性强，接近自然语言",
        "icon": "check-circle"
      },
      {
        "id": "point-2",
        "type": "key-point",
        "title": "应用广泛",
        "content": "Web开发、AI、数据分析、自动化等领域",
        "icon": "globe"
      },
      {
        "id": "point-3",
        "type": "key-point",
        "title": "生态丰富",
        "content": "海量第三方库和框架，开发效率高",
        "icon": "package"
      },
      {
        "id": "point-4",
        "type": "key-point",
        "title": "跨平台",
        "content": "支持Windows、Mac、Linux，一次编写到处运行",
        "icon": "layers"
      }
    ]
  }
}
```

### 布局特点
- **标题区**：顶部居中，大字体醒目
- **核心观点区**：网格布局，每个要点带图标
- **详细说明区**：列表或卡片式展示
- **总结区**：底部强调关键要点

### 风格推荐
- **技术内容**：极简风、商务风
- **生活内容**：马卡龙风、手绘风
- **科普内容**：极简风、扁平化

### 使用示例

**输入**：
```
生成Python编程语言的信息图，包含4个核心特点：
1. 语法简洁 - 易学易用
2. 应用广泛 - Web、AI、数据分析
3. 生态丰富 - 海量库和框架
4. 跨平台 - Windows、Mac、Linux
```

**输出**：极简风格的知识科普信息图

## 2. 对比分析模板（comparison）

### 适用场景
- 产品对比
- 方案评估
- 优劣分析
- Before/After对比
- 竞品分析

### 内容结构

```
标题区
├─ 主标题（必填）
└─ 对比对象说明（可选）

对比区（必填）
├─ 对象A（左侧/上方）
│   ├─ 名称
│   ├─ 要点1
│   ├─ 要点2
│   └─ ...
└─ 对象B（右侧/下方）
    ├─ 名称
    ├─ 要点1
    ├─ 要点2
    └─ ...

结论区（可选）
├─ 对比总结
└─ 选择建议
```

### JSON示例

```json
{
  "metadata": {
    "template": "comparison",
    "style": "business"
  },
  "content": {
    "title": {
      "text": "Python vs JavaScript",
      "subtitle": "两种主流编程语言对比"
    },
    "sections": [
      {
        "id": "python",
        "type": "comparison-item",
        "title": "Python",
        "content": "",
        "items": [
          {"label": "应用领域", "value": "AI、数据分析、后端"},
          {"label": "学习难度", "value": "简单"},
          {"label": "运行速度", "value": "较慢"},
          {"label": "生态", "value": "丰富（科学计算）"}
        ]
      },
      {
        "id": "javascript",
        "type": "comparison-item",
        "title": "JavaScript",
        "content": "",
        "items": [
          {"label": "应用领域", "value": "前端、全栈、移动端"},
          {"label": "学习难度", "value": "中等"},
          {"label": "运行速度", "value": "快"},
          {"label": "生态", "value": "丰富（Web开发）"}
        ]
      }
    ],
    "conclusion": {
      "text": "Python适合AI和数据科学，JavaScript适合Web开发",
      "recommendation": "根据项目需求选择合适的语言"
    }
  }
}
```

### 布局特点
- **标题区**：顶部居中
- **对比区**：左右分栏或上下分区
- **要点对齐**：相同维度的要点对齐展示
- **视觉对比**：使用颜色、图标区分优劣

### 风格推荐
- **商务对比**：商务风
- **产品对比**：扁平化
- **技术对比**：极简风

### 使用示例

**输入**：
```
对比Python和JavaScript编程语言，从应用领域、学习难度、运行速度、生态4个维度分析
```

**输出**：左右分栏的对比分析信息图

## 3. 流程说明模板（process）

### 适用场景
- 步骤指南
- 操作流程
- 工作流
- 教程说明
- SOP标准流程

### 内容结构

```
标题区
├─ 主标题（必填）
└─ 流程简介（可选）

流程区（必填，3-10个步骤）
├─ 步骤1
│   ├─ 标题
│   ├─ 描述
│   ├─ 关键点（可选）
│   └─ 图标
├─ 步骤2
├─ 步骤3
└─ ...

注意事项区（可选）
├─ 注意点1
├─ 注意点2
└─ ...

总结区（可选）
└─ 流程要点回顾
```

### JSON示例

```json
{
  "metadata": {
    "template": "process",
    "style": "flat"
  },
  "content": {
    "title": {
      "text": "Git工作流程",
      "subtitle": "标准的Git协作流程"
    },
    "sections": [
      {
        "id": "step-1",
        "type": "step",
        "title": "克隆仓库",
        "content": "使用git clone命令克隆远程仓库到本地",
        "icon": "download",
        "keyPoints": ["确保仓库地址正确", "检查权限"]
      },
      {
        "id": "step-2",
        "type": "step",
        "title": "创建分支",
        "content": "创建新的功能分支进行开发",
        "icon": "git-branch",
        "keyPoints": ["分支命名规范", "从最新代码创建"]
      },
      {
        "id": "step-3",
        "type": "step",
        "title": "提交代码",
        "content": "完成开发后提交代码到本地仓库",
        "icon": "git-commit",
        "keyPoints": ["写清楚commit message", "提交前自测"]
      },
      {
        "id": "step-4",
        "type": "step",
        "title": "推送分支",
        "content": "推送分支到远程仓库",
        "icon": "upload",
        "keyPoints": ["确认目标分支", "解决冲突"]
      },
      {
        "id": "step-5",
        "type": "step",
        "title": "创建PR",
        "content": "在远程仓库创建Pull Request",

      "及时同步远程分支变更",

    ]
  }
}
```

### 布局特点
- **流程区**：水平或垂直流程图
- **步骤连接**：箭头或连线
- **步骤编号**：清晰的步骤序号
- **关键点标注**：小提示框

### 风格推荐
- **操作流程**：扁平化
- **工作流程**：商务风
- **教程指南**：手绘风

### 使用示例

**输入**：
```
创建Git工作流程信息图，包含5个步骤：
克隆仓库 → 创建分支 → 提交代码 → 推送分支 → 创建PR
每个步骤要有标题、描述和关键点
```

**输出**：扁平化风格的流程说明信息图

## 4. 数据展示模板（data）

### 适用场景
- 数据报告
- 统计结果
- 调研分析
- KPI展示
- 数据可视化

### 内容结构

```
标题区
├─ 主标题（必填）
└─ 数据来源说明（可选）

图表区（必填）
├─ 图表类型（柱状图/折线图/饼图等）
├─ 数据
└─ 图例

数据说明区（必填）
├─ 关键数据解读
├─ 数据洞察
└─ 对比分析

结论区（可选）
├─ 主要发现
└─ 行动建议
```

### JSON示例

```json
{
  "metadata": {
    "template": "data",
    "style": "business"
  },
  "content": {
    "title": {
      "text": "2024年Q1销售数据分析",
      "subtitle": "三大产品线销售表现对比"
    },
    "sections": [
      {
        "id": "chart-1",
        "type": "chart",
        "chartType": "bar",
        "title": "各产品线销售额对比",
        "data": {
          "labels": ["产品A", "产品B", "产品C"],
          "datasets": [
            {
              "label": "Q1销售额（万元）",
              "data": [120, 85, 65],
              "backgroundColor": ["#1976d2", "#4caf50", "#ff9800"]
            }
          ]
        }
      }
    ],
    "insights": [
      "产品A表现最佳，占总额43%",
      "产品B环比增长15%",
      "产品C需要加强市场推广"
    ],
    "conclusion": {
      "findings": "产品A是主力产品，产品B增长潜力大",
      "actions": "加大产品B投入，优化产品C定位"
    }
  }
}
```

### 布局特点
- **图表区**：居中展示，清晰易读
- **数据标注**：关键数据突出显示
- **对比视图**：支持多维度对比
- **图例说明**：明确的图例和单位

### 风格推荐
- **商务报告**：商务风
- **数据科学**：极简风
- **科研论文**：极简风（学术风格）

### 使用示例

**输入**：
```
生成2024年Q1销售数据分析图表，对比产品A（120万）、产品B（85万）、产品C（65万）的销售额
```

**输出**：柱状图展示的数据分析信息图

## 6. 科研图表模板（scientific）

### 适用场景
- 学术论文
- 研究报告
- 实验数据
- 科研成果展示
- 学术会议

### 内容结构

```
标题区（必填）
├─ 标题（学术规范）
└─ 作者/单位信息（可选）

图表区（必填）
├─ 标准图表类型
├─ 完整坐标轴标注
├─ 图例
└─ 数据标注

数据说明区（必填）
├─ 方法说明
├─ 数据来源
└─ 参数说明

引用区（必填）
├─ 参考文献
└─ 数据来源标注
```

### JSON示例

```json
{
  "metadata": {
    "template": "scientific",
    "style": "minimal"
  },
  "content": {
    "title": {
      "text": "三种排序算法时间复杂度对比",
      "subtitle": "实验数据：n=10000"
    },
    "sections": [
      {
        "id": "chart-1",
        "type": "chart",
        "chartType": "bar",
        "title": "平均执行时间对比",
        "data": {
          "labels": ["快速排序", "归并排序", "堆排序"],
          "datasets": [
            {
              "label": "执行时间（ms）",
              "data": [12.5, 15.2, 18.3],
              "backgroundColor": "#1976d2"
            }
          ]
        },
        "axisLabels": {
          "x": "排序算法",
          "y": "执行时间（毫秒）"
        }
      }
    ],
    "methods": {
      "description": "在相同硬件环境下测试三种排序算法",
      "parameters": {
        "dataSize": "n=10000",
        "testRounds": "100次取平均",
        "hardware": "Intel i7-12700K, 32GB RAM"
      }
    },
    "references": [
      "[1] Cormen, T. H. et al. Introduction to Algorithms. MIT Press, 2009.",
      "[2] 数据来源：自测数据"
    ]
  }
}
```

### 布局特点
- **学术规范**：符合学术论文标准
- **图表清晰**：高分辨率，适合打印
- **标注完整**：坐标轴、图例、单位
- **引用规范**：参考文献格式正确
- **中性配色**：黑白灰为主，适合打印

### 风格推荐
- **科研图表**：极简风（学术风格）
- **数据可视化**：极简风

### 科研特性

**内容要求**：
1. 数据准确：所有数据有明确来源
2. 逻辑严谨：论证过程完整
3. 专业术语：使用行业标准术语
4. 引用规范：标注参考文献

**视觉规范**：
1. 图表标准：使用标准科研图表类型
2. 坐标标注：完整的坐标轴说明
3. 图例清晰：明确的图例和标注
4. 分辨率足够：300dpi以上，适合学术发表

**推荐图表类型**：
- 数据对比：柱状图、折线图
- 比例关系：饼图、环形图
- 分布情况：散点图、箱线图
- 相关性：散点图、热力图
- 流程关系：流程图、网络图

### 使用示例

**输入**：
```
制作三种排序算法性能对比图表：
快速排序（12.5ms）、归并排序（15.2ms）、堆排序（18.3ms）
科研风格，包含坐标轴标注、方法说明、参考文献
```

**输出**：符合学术规范的科研图表

## 自定义模板

### 创建自定义模板

1. **创建模板目录**：
```bash
mkdir -p assets/templates/my-template
```

2. **创建模板配置** `template.json`：
```json
{
  "name": "my-template",
  "description": "自定义模板描述",
  "layout": {
    "type": "vertical" | "horizontal" | "grid",
    "sections": ["section1", "section2", ...],
    "spacing": 40
  },
  "defaults": {
    "dimensions": {
      "width": 1920,
      "height": 1080
    },
    "style": "minimal"
  },
  "validation": {
    "required": ["title", "sections"],
    "minSections": 1,
    "maxSections": 10
  }
}
```

3. **创建布局定义** `layout.json`：
```json
{
  "title": {
    "position": "top",
    "alignment": "center",
    "style": {...}
  },
  "sections": {
    "layout": "grid",
    "columns": 2,
    "gap": 20
  }
}
```

4. **创建风格变体** `styles/*.json`：
```json
{
  "name": "minimal",
  "colors": {...},
  "typography": {...}
}
```

5. **注册模板**：

在`renderer-config.json`中添加：
```json
{
  "templates": {
    "my-template": {
      "path": "assets/templates/my-template",
      "enabled": true
    }
  }
}
```

### 使用自定义模板

```bash
node skill-render.js "生成信息图" --template my-template
```

## 模板最佳实践

### 内容准备

1. **明确主题**：确定信息图的核心主题
2. **收集要点**：列出所有要展示的关键信息
3. **组织结构**：按照模板结构组织内容
4. **准备数据**：准备好图表数据（如适用）

### 风格选择

1. **匹配受众**：风格要符合受众偏好
2. **匹配内容**：风格要匹配内容类型
3. **保持一致**：同系列信息图使用相同风格
4. **考虑平台**：不同平台推荐不同风格

### 质量检查

生成后检查：
- ✅ 内容完整性：所有要点都展示了
- ✅ 逻辑清晰：信息层次清晰
- ✅ 视觉美观：颜色、排版协调
- ✅ 易于理解：受众能快速理解
- ✅ 文件质量：分辨率、清晰度足够

### 迭代优化

1. **收集反馈**：向目标受众收集意见
2. **分析数据**：查看阅读量、互动数据
3. **持续改进**：根据反馈优化内容和设计
4. **A/B测试**：尝试不同风格，选择最佳
